Build a full-stack web application named **“Project Hub – Centralized Student Project Management System”** using the following requirements.

## 🧠 Overview

The system should allow students and admins to manage academic projects in a centralized platform. It should include authentication, role-based dashboards, project tracking, and profile management.

---

## ⚙️ Tech Stack

* Backend: Python (Flask)
* Frontend: HTML, CSS, Bootstrap
* Database: SQLite (using SQLAlchemy ORM)

---

## 🔐 Authentication Flow

1. Role Selection Page:

   * User selects either "Student" or "Admin"

2. Signup:

   * Fields: Email, Username, Password
   * After signup, generate a 6-digit OTP
   * Send OTP to user's email (use SMTP; if not possible, print OTP in console)

3. OTP Verification Page:

   * User enters OTP
   * If valid → proceed to Personal Details page

4. Personal Details Page:

   * Fields:

     * Name
     * Mobile Number
     * Date of Birth
     * College Name
     * Department (Dropdown: CSE, IT, AIDS, ECE, EEE, MECH)
     * Year (Dropdown: 1st, 2nd, 3rd, 4th)
   * Save all details to database

5. Login:

   * Login using email/username and password
   * Redirect based on role:

     * Student → Student Dashboard
     * Admin → Admin Dashboard

---

## 👤 Student Features

After login, student should have:

1. Dashboard:

   * Show total projects, completed, pending

2. My Projects:

   * View all projects added by the student

3. Add New Project:

   * Fields: Title, Description
   * Save to database

4. Project Updates:

   * Add daily updates for each project
   * View updates

5. Profile:

   * Display personal details
   * Option to edit

6. Logout

---

## 👨‍💼 Admin Features

1. Dashboard:

   * Total users
   * Total projects

2. Manage Projects:

   * View all student projects
   * Update project status (approve/reject/complete)

3. Manage Users:

   * View all users
   * Delete users if needed

4. Logout

---

## 🛢️ Database Design

### Table: Users

* id (Primary Key)
* email
* username
* password
* role (student/admin)
* name
* mobile
* dob
* college
* department
* year

### Table: Projects

* id
* title
* description
* status (Pending/Completed)
* student_id (Foreign Key)
* created_at

### Table: ProjectUpdates

* id
* project_id (Foreign Key)
* update_text
* date
